# プロジェクト概要

プロジェクト概要には、「coachtech勤怠管理アプリ」の開発計画を記載する。

---

## 1. Web サービス制作の概要・方針

- **サービス名:** coachtech勤怠管理アプリ
- **サービス概要:** ある企業が開発した独自の勤怠管理アプリ
- **制作の背景と目的:** ユーザーの勤怠と管理を目的とする
- **制作の目標:** 初年度でユーザー数1,000人達成
- **作業範囲:** 設計、コーディング、テスト
- **納品方法:** GitHubでのリポジトリ共有

---

## 2. 業務要件

- **現行サービスの分析:** 新規サービスの為、考慮しない
- **競合他社の調査・分析:** 機能や画面が複雑で使いづらい

---

## 3. サイト要件

- **ターゲットユーザー:** 社会人全般
- **ターゲットブラウザ・OS:** PC: Chrome/Firefox/Safariの最新バージョン
- **画面一覧:** [画面設計](ui_design.md)に記載

---

## 4. システム要件

### 機能要件一覧

[機能要件](functional_requirements.md)に記載

### 非機能要件

- **運用・保守:** クライアントが運用・保守を行う
- **リリース:** 4ヶ月後を予定
- **セキュリティ:** アプリケーション内に限り考慮する
- **SEO:** 考慮しない
- **コード品質:** [コーディング規約](https://estra-inc.notion.site/1263a94a2aab4e3ab81bad77db1cf186)を参照してコーディングを行う事、また、その他の要件については[開発プロセス](development_process.md)を参照する事

### インフラ要件

- **本番環境・開発環境:** 開発環境 (ローカル環境)は開発者が用意
- **サーバー:** サーバーは設置しない
- **ドメイン:** ドメインは取得しない
- **SSL化:** SSL化は考慮しない
- **データベース:** MySQL

### デザイン要件

- **UIデザイン:** [画面設計](ui_design.md)に記載

### テスト計画

- **テスト項目:** [開発プロセス](development_process.md)に記載
- **テスト要件:** [テストケース一覧](test_cases.md)に記載

### 開発手法

- **開発言語:** PHP
- **フレームワーク:** Laravel
- **バージョン管理:** Docker, GitHub

---

## 5. その他

- **スケジュール:** 担当コーチと相談した期間で実装する事 (2026/03/07 - 2026/05/13)
- **プロジェクト開発体制:** 開発者が1人の為、作成しない
- **使用用語の定義:** 特別な用語はない為、作成しない
